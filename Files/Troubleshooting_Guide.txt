AUTO DIAG PRO - TROUBLESHOOTING GUIDE
=====================================

This guide helps you resolve common issues with AutoDiag Pro.

ISSUE 1: "Python is not recognized"
-----------------------------------
SYMPTOMS: Command prompt says "python is not recognized"
SOLUTION:
1. Reinstall Python and check "Add Python to PATH"
2. Or manually add Python to PATH:
   - Search for "Environment Variables"
   - Edit "Path" in System Variables
   - Add: C:\Python3x\ and C:\Python3x\Scripts\

ISSUE 2: "Module not found" errors
-----------------------------------
SYMPTOMS: Import errors for PyQt6, serial, can, obd
SOLUTION:
1. Open Command Prompt as Administrator
2. Run: pip install PyQt6 pyserial python-can obd
3. If pip fails: python -m pip install --upgrade pip

ISSUE 3: Application won't start
---------------------------------
SYMPTOMS: Launcher opens but closes immediately
SOLUTION:
1. Check launcher_debug.log for error messages
2. Ensure all required packages are installed
3. Try running: python launcher.py directly

ISSUE 4: Database connection errors
-----------------------------------
SYMPTOMS: "Cannot connect to database" or SQL errors
SOLUTION:
1. Install SQL Server Express from Microsoft
2. Or use the bundled installer during setup
3. Check if SQL Server service is running

ISSUE 5: VCI device not detected
--------------------------------
SYMPTOMS: "No VCI device found" or connection errors
SOLUTION:
1. Install VCI drivers from the drivers folder
2. Try different USB ports
3. Check device manager for driver issues
4. Restart computer after driver installation

ISSUE 6: Permission denied errors
---------------------------------
SYMPTOMS: "Access denied" or "Permission denied"
SOLUTION:
1. Run AutoDiag Pro as Administrator
2. Check antivirus/firewall settings
3. Ensure user has write permissions to installation folder

LOG FILES
---------
Check these files for detailed error information:
- launcher_debug.log (main launcher errors)
- AutoDiag/logs/ (application logs)
- Windows Event Viewer (system errors)
